# 方法摘要

本项目采用“约束优先、分题建模”的混合流水线。正式结果、算法基线、参考答案诊断和进阶实验分别存放，不以单一分数掩盖数据边界差异。

## A. 描原：距离场几何先验 + LOFO 适配

**目标。** 将彩色粗稿转换为白/黑/蓝/红四种精确色组成的规范线稿，并控制线宽、短裂缝和闭合性。

**流程。**

1. 将粗稿转为亮度/色度特征，检测结构线与红/蓝语义线；
2. 研究分支使用 Deep Sketch Vectorization 风格的中心线/截断距离场表征；
3. 在三张 KTK_04 成对帧上执行三折 leave-one-frame-out 适配，每折只用另外两张成品；
4. 以确定性融合得到留出帧输出；
5. 仅执行保守闭运算、方向一致短缝连接和场记屏蔽；
6. 将像素量化为题面规定的四种精确 RGB。

参考成品中存在额外绿色生产语义线，因此另提供五色补充版；绿色仅从当前粗稿的高饱和绿色核心恢复，不读取当前帧成品。A 研究 checkpoint 的外部预训练语料许可链不完整，故标为 research-only；CPU 规则基线不依赖这些权重。

## B. 中割：距离场双向运动 + 像素图拓扑正则

**目标。** 根据 A1/A6/A9 与律表生成 A2--A5、A7--A8，保持线色、结构和序列连续性。

**流程。**

1. 将端点线稿转为截断距离场；
2. 使用双向 DIS 光流估计粗运动；
3. 在端点/交点之间提取有序笔画，对位移做二阶平滑；
4. 将正则化运动作用于八连通像素图边，避免直接 RGB 混合产生模糊；
5. 对红/蓝/绿稀疏语义线分别估计运动，避免黑线支配；
6. A1--A6 使用双向插值，A6--A9 大转头使用目标关键帧拓扑的反向场。

正式 blind 版本不加载任何由缺失帧成品训练的权重。`b_cleanup.pth` 仅使用 A2--A5 成品训练，且只进入 `outputs/diagnostic/task_b_shot_adapted`。其 A7/A8 F1 略高，但线量约为参考的 2.2 倍，不满足生产线稿约束。

## C. 上色：区域分解 + 关键视图语义 + 包含传播

**目标。** 使用设定图精确色值在封闭区域内平涂，并保证输入非白线逐像素不变。

**流程。**

1. 从角色设定图固定色卡坐标提取精确 RGB 调色盘；
2. 对每帧输入线稿做四连通白区分解；
3. 零标签基线以面积、位置和设定色阶为高置信大区赋色；
4. 正式配置在 A1/A5/A7/A9 四个关键视图建立 153 个 paint-bucket 标签；
5. 通过距离场运动、区域包含率和多参考投票向相邻帧传播；
6. 冲突或低置信区域留白；输出从输入副本开始，只修改原白像素。

225 标签 dense-assisted 和 oracle/reviewed 配置包含参考答案审计后的补点，仅用于诊断信息上限，不计入正式成绩。

## 统一评测

线稿使用 exact/1px/2px 双向容差 F1、逐色 F1、色规与闭合泄漏；上色使用面积精度、正确覆盖、区域精度/覆盖与线像素变化。`tools/validate_submission.py` 从原始素材重新计算 JSON 和 CSV 指标并 fail-closed 退出。
