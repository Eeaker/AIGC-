# 数据与评测边界

| 项目 | 生成时可用 | 仅评测时打开 | 不作声明 |
|---|---|---|---|
| A 正式 | 三张粗稿；每折另外两张成对图；外部几何数据 | 当前留出帧成品 | 跨镜头通用描原模型 |
| B 正式 blind | A1/A6/A9 端点与律表；不加载成品监督权重 | A2-A5/A7/A8 成品仅评分 | 无监督或跨镜头泛化 |
| B shot-adapted 诊断 | blind 输出；A2-A5 成品训练的清理权重 | A7/A8 成品仅评分 | 正式成绩、通用模型 |
| B 标注 v2 | A1/A6/A9 端点、人工点/边标签 | A7/A8 成品仅在渲染后评分 | 完整人工 GT、训练出的泛化 visibility 模型 |
| C 正式 | 九张输入线稿、设定图/色卡、153 个干净关键视图区域标签 | 九张成品仅用于最后评分 | 全自动上色 |
| C automatic | 九张输入线稿、设定图规则；0 个语义标签 | 九张成品仅评分 | 高覆盖生产结果 |
| C oracle/reviewed | 研究阶段参考答案误差审计 | 同左 | 正式结果 |

A/B/C 的数值均由同一独立脚本重算。A/B 同时报告 exact、1 px、2 px F1 和逐色指标，防止容差掩盖错色或粗线；B 还比较线量与连通组件。C 以输入线稿的四连通白区统计精度/覆盖，并单独断言所有输入非白像素不变。

标注包的 `validation_report.json` 为结构合法性验证，不等价于人工正确性证明。当前 v2 中 871 个可见性事件仍需复核，只有 24 条边可见性记录，因此它适合做稀疏约束消融，不足以监督通用模型。
