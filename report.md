## 2. A 描原：四色正式版与绿色生产补充版

几何分支采用外部中心线/截断距离场预训练，再对 KTK_04 三张成对数据做三折 LOFO：每折只用另外两张成品适配，留出帧参考在推理后评分。后处理只做保守 3x3 闭运算和方向一致的短缝连接，防止凭空新增结构。

题面明确要求白/黑/蓝/红四种精确色，故 `outputs/official/task_a` 为正式结果。三张参考成品又确实存在绿色语义线；直接删除会降低生产一致性，因此另提供五色补充版。五色版从当前粗稿高饱和绿色核心恢复，不读取成品，平均 F1@2px 约 0.9913；早期强制单像素骨架使绿线量只剩参考的 51%-60%，已经废弃。

进阶 KTK_05 同时提交 A/B 层：A 层严格四色两帧 F1@2px 约 0.939；B 是稀疏嘴部修正层，角色 ROI F1 为 0.856/1.000。它验证了冻结规则能迁移，但样本规模不足以宣称通用模型。

## 3. B 中割：像素图拓扑变形与失败反证

端点线稿先转截断距离场，以双向 DIS 估计粗运动；沿端点/交点之间的有序笔画正则位移，再直接变形八连通像素图的边。A6-A9 大转头采用 A0009 单拓扑反向场，红/绿稀疏线独立估计。正式 blind 输出不加载由 A2-A5 成品训练的 cleanup 权重；该 shot-adapted 版本只放诊断目录。律表时刻为 A2-A5 的 0.25/0.50/0.75/0.90 与 A7/A8 的 0.50/0.75。

新 topology_visibility_v2 包结构验证通过，含 33 个 A6-A9 人工语义点、24 条边可见性记录；但 871 个 visibility 事件仍待复核，所以只能称保守预标注。全局 TPS 的 A7/A8 最佳仅约 0.209/0.240，部件图直接形变更低至 0.165/0.230；局部残差融合平均 F1@2px 仅 +0.00010。预算化 Assisted-10 从 0.55303 到 0.55390，未过预先规定的 +0.002 多指标接受阈值，Assisted-30 也未胜出，均不替换正式方案。

云端候选同样按多指标淘汰：RAFT/MIBA 近似仅 0.201/0.346；学习清理虽到 0.544/0.593，却把线量放大到 2.19/2.33 倍。soft-clDice 及若干论文实现的历史试验因本包证据链不完整，不再报告精确数值。结论是瓶颈为发丝出生/消失、遮挡和笔画对应，不是缺 GPU。只有获得多镜头、完成复核的端点/交点/visibility 数据后，训练图匹配模型才有意义。

## 4. C 上色、失败分析与 AI 使用

从设定图精确统计 RGB 调色盘，对每帧输入做四连通白区分解。正式 153 个 paint-bucket 标签描述四个关键视图 A1/A5/A7/A9，颜色只来自线稿、角色设定图和色卡，不使用 A2-A8 中间成品误差修补。区域 mask 通过距离场运动和 inclusion 投票传播；冲突区域保守留白。输出从输入副本开始，所有非白线像素逐点不变。

五档对照清楚说明人工收益：automatic 正确覆盖 59.25%；正式 153 点 assisted 95.63%；225 点 dense-assisted 99.30%；oracle/reviewed 可到约 99.94%。后两者含成品对照后追加标签或误差审计，只保留为诊断，不作为正式成绩。剩余留白主要集中于 A1-A6 的低置信边界连通衣物区域，以及少量眼部、发丝微小白岛；历史邻域盲填准确率仅约 35.8%，清零会制造更多错色。

本轮未可靠记录实际标注时长，因此只报告 153 个区域标签数量，不将估算时间作为实验结论。

![题 C 对比](outputs/comparisons/task_c.png)

KTK_05 进阶提交严格保线 A/B 层并评测阴影/主体/高光。A 层用 A1/A3/A5 成品作为明确训练关键帧，A2/A4 成品仅评测；修复凹区域质心错绑 label 与错误关键帧传播后，A1-A5 已填区域精度为 99.15%/98.91%/100.00%/98.73%/99.57%，覆盖为 88.61%/85.36%/69.71%/58.04%/63.64%，原线像素变化均为 0。低置信区域依题面“错涂重于留白”保留白色，不把强行填满冒充质量提升；B2/B3 精度与覆盖约 100.00%/93.87%。

主要改进方向：（1）A 在授权的多镜头四色 rough-clean 对上按镜头划分训练/验证/测试；（2）B 完成 visibility 标注复核，学习笔画图对应并用线量/组件/F1 联合选模；（3）C 建立区域邻接与包含图，只在双向一致时自动填色并记录人工分钟数。

使用 ChatGPT/Codex 辅助题面拆解、实现、像素审计、论文/作者实现调研、AutoDL 实验和报告，估计约 20-40 万 token，主要消耗在 B 反证、A 三折消融和 C 区域审计。RTX 4090 用于 A 预训练/LOFO 与 B 消融；金额及累计时长未可靠记录。主要参考同行评审或作者一手资料：Deep Sketch Vectorization（SIGGRAPH 2024）、AnimeInbet（ICCV 2023）、MIBA（ACCV 2024）、TPS-Inbetween、JoSTC（SIGGRAPH 2024）、Learning Inclusion Matching（CVPR 2024）。第三方研究权重未进入正式交付。
